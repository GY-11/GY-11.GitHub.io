## 算法逻辑说明

### DINA模型核心原理
DINA（Deterministic Input, Noisy "And" Gate）模型是一种认知诊断模型，其核心原理是：**学生对题目的作答概率，取决于是否掌握了该题考察的所有必备知识点**。

### 算法步骤
1. **数据准备**：
   - Q矩阵：题目-知识点关联矩阵，记录每个题目考察的知识点
   - X矩阵：学生-题目作答矩阵，记录学生对每个题目的作答结果

2. **参数设置**：
   - 失误率（s）：掌握所有知识点仍答错的概率，默认值为0.1
   - 猜测率（g）：未掌握所有知识点仍答对的概率，默认值为0.2

3. **掌握概率计算**：
   - 对每个知识点，找出所有考察该知识点的题目
   - 统计学生在这些题目上的作答情况（正确数和总题数）
   - 使用极大似然估计计算掌握概率

4. **计算公式**：
   - 似然函数：
     - P(correct|mastered) = (1-s)^correct * s^(1-correct)
     - P(correct|not mastered) = g^correct * (1-g)^(1-correct)
   - 后验概率：
     - P(mastered|correct) = P(correct|mastered) / [P(correct|mastered) + P(correct|not mastered)]

5. **结果输出**：
   - 返回学生在所有知识点上的掌握概率字典

### 算法特点
- **简单高效**：计算逻辑清晰，易于实现
- **可解释性强**：结果直观易懂，便于教师和学生理解
- **适应性好**：可应用于不同学科和知识点体系
- **鲁棒性**：通过设置合理的s和g参数，能够处理一定的作答噪声

### 应用场景
- **个性化学习**：根据学生的知识掌握情况，推荐针对性的学习内容
- **教学评估**：评估教学效果，发现学生的知识薄弱点
- **学习路径规划**：基于知识图谱和掌握情况，规划最优学习路径

### 代码实现
DINA模型的核心实现位于`backend/model/dina.py`文件中，主要包括：
- `__init__`方法：初始化模型参数（s和g）
- `predict`方法：计算学生知识点掌握概率
- 知识点提取、相关题目统计、掌握概率计算等核心逻辑
